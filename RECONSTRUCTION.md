# Reconstruction rules

This repository records a gradual reconstruction of engineering projects from memory.

## Evidence levels

Each technical statement should be treated as one of:

- **Confirmed memory** — remembered clearly by the original author.
- **Probable reconstruction** — consistent with the remembered design, but not yet confirmed.
- **Open question** — a detail that remains uncertain.
- **Documentary evidence** — supported by surviving code, diagrams, photographs, documents, or hardware.

Do not silently replace an earlier recollection when memory changes. Record the correction and explain what changed.

## Working method

1. Raw recollections go to `vault/inbox.md`.
2. Unresolved details go to `vault/questions.md`.
3. Dates and milestones go to `vault/timeline.md`.
4. Confirmed material is gradually incorporated into project documents.
5. Editorial text must distinguish remembered facts from later interpretation.
